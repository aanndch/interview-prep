The Binary Search Framework

Binary Search is a divide-and-conquer paradigm that operates on a sorted search space. By examining the middle index, we can eliminate half of the potential candidates in O(1) decision time. To set up standard Binary Search:

  • Indices: Initialize `L = 0` and `R = length - 1`.
  • Loop Condition: Loop while `L <= R`.
  • Midpoint Guard: Avoid integer overflow by calculating mid as `L + Math.floor((R - L) / 2)`.

#704 Binary Search

Easy
Given an array of integers nums which is sorted in ascending order, and an integer target, write a function to search target in nums. If target exists, then return its index. Otherwise, return -1.

The array is sorted. Instead of checking every index (O(N)), we check the middle element `nums[mid]`.

If `target` matches `nums[mid]`, return its index. If `target` is larger, we look in the right half by setting `L = mid + 1`. If `target` is smaller, search the left half by setting `R = mid - 1`.

Input: nums = [-1, 0, 3, 5, 9, 12], target = 9 Step 1: L = 0, R = 5 - mid = floor((0 + 5) / 2) = 2 - nums[mid] = nums[2] = 3 - 3 < 9 -> Target in right half. Set L = mid + 1 = 3. Step 2: L = 3, R = 5 - mid = floor((3 + 5) / 2) = 4 - nums[mid] = nums[4] = 9 - 9 === 9 -> Match! Return index 4.
TypeScript 
function search(nums: number[], target: number): number {
    let l = 0;
    let r = nums.length - 1;
    
    while (l <= r) {
        const mid = l + Math.floor((r - l) / 2);
        if (nums[mid] === target) {
            return mid;
        } else if (nums[mid] < target) {
            l = mid + 1;
        } else {
            r = mid - 1;
        }
    }
    return -1;
}
Time Complexity: O(log N)
Space Complexity: O(1)

#35 Search Insert Position

Easy
Given a sorted array of distinct integers and a target value, return the index if the target is found. If not, return the index where it would be if it were inserted in order.

This is a binary search to find the insert index. When the binary search finishes without finding the target, the left pointer `L` points to the correct insertion index.

Input: nums = [1, 3, 5, 6], target = 5 Step 1: L = 0, R = 3 - mid = floor((0 + 3) / 2) = 1 - nums[mid] = nums[1] = 3 - 3 < 5 -> Set L = mid + 1 = 2 Step 2: L = 2, R = 3 - mid = floor((2 + 3) / 2) = 2 - nums[mid] = nums[2] = 5 - 5 === 5 -> Match! Return index 2.
TypeScript 
function searchInsert(nums: number[], target: number): number {

    let r = nums.length - 1;

            return mid;

            l = mid + 1;
        } else {
            r = mid - 1;
        }

}

Space Complexity: O(1)

#69 Sqrt(x)

Easy
Given a non-negative integer x, return the square root of x rounded down to the nearest integer. The returned integer should be non-negative as well.

The square root of `x` must reside between `1` and `x`. Our search space is therefore sorted.

Calculate `mid * mid`. If it matches `x`, `mid` is the answer. If it is smaller than `x`, store `mid` as a potential answer and narrow the bounds to the right side (to look for a larger value). If larger, search the left half.

Input: x = 8 Search space: L = 1, R = 8 Step 1: L = 1, R = 8 - mid = floor((1 + 8) / 2) = 4 - 4 * 4 = 16 > 8 -> Set R = mid - 1 = 3 Step 2: L = 1, R = 3 - mid = floor((1 + 3) / 2) = 2 - 2 * 2 = 4 <= 8 -> Potential match. Store res = 2. Set L = mid + 1 = 3 Step 3: L = 3, R = 3 - mid = floor((3 + 3) / 2) = 3 - 3 * 3 = 9 > 8 -> Set R = mid - 1 = 2 Step 4: L = 3, R = 2 - L > R -> Loop terminates. Return res = 2.
TypeScript 
function mySqrt(x: number): number {
    if (x < 2) return x;
    let l = 1;
    let r = Math.floor(x / 2);
    let res = 0;
    
    while (l <= r) {
        const mid = l + Math.floor((r - l) / 2);
        const sqr = mid * mid;
        
        if (sqr === x) {
            return mid;
        } else if (sqr < x) {
            res = mid;
            l = mid + 1;
        } else {
            r = mid - 1;
        }
    }
    return res;
}
Time Complexity: O(log X)
Space Complexity: O(1)

#278 First Bad Version

Easy
You are a product manager and currently leading a team to develop a new product. Unfortunately, the latest version of your product fails the quality check. Since each version is developed based on the previous version, all the versions after a bad version are also bad.

Suppose you have n versions [1, 2, ..., n] and you want to find out the first bad one, which causes all the following ones to be bad.

The versions are sorted sequentially. If version `mid` is bad, all versions after it are also bad, which means the first bad version lies at or before `mid`. Narrow the search space to the left by setting `R = mid`.

If version `mid` is good, all versions before it are also good. Search the right half by setting `L = mid + 1`. Loop terminates when `L === R`.

Input: n = 5, bad = 4 (Versions: 1, 2, 3, 4, 5) Step 1: L = 1, R = 5 - mid = floor((1 + 5) / 2) = 3 - isBadVersion(3) = false -> search right half. Set L = mid + 1 = 4. Step 2: L = 4, R = 5 - mid = floor((4 + 5) / 2) = 4 - isBadVersion(4) = true -> Potential solution. Set R = mid = 4. Step 3: L = 4, R = 4 - L === R -> Loop terminates. First bad version = 4.
TypeScript 
// Boilerplate wrapper for API
var solution = function(isBadVersion: any) {
    return function(n: number): number {
        let l = 1;
        let r = n;
        
        while (l < r) {
            const mid = l + Math.floor((r - l) / 2);
            if (isBadVersion(mid)) {
                r = mid;
            } else {
                l = mid + 1;
            }
        }
        return l;
    };
};
Time Complexity: O(log N)
Space Complexity: O(1)

#153 Find Minimum in Rotated Sorted Array

Medium
Suppose an array of length n sorted in ascending order is rotated between 1 and n times.

Given the sorted rotated array nums of unique elements, return the minimum element of this array.

In a sorted rotated array, compare `nums[mid]` with `nums[r]`.

If `nums[mid] > nums[r]`, it indicates the pivot (minimum element) lies to the right of `mid`. Shift the left boundary past `mid` (`L = mid + 1`). Otherwise, the minimum element sits at or before `mid` (`R = mid`).

Input: nums = [3, 4, 5, 1, 2] Step 1: L = 0, R = 4 - mid = floor((0 + 4) / 2) = 2 - nums[mid] = nums[2] = 5 - nums[mid] > nums[R] (5 > 2) -> Minimum lies to the right. Set L = mid + 1 = 3. Step 2: L = 3, R = 4 - mid = floor((3 + 4) / 2) = 3 - nums[mid] = nums[3] = 1 - nums[mid] <= nums[R] (1 <= 2) -> Minimum could be mid itself. Set R = mid = 3. Step 3: L = 3, R = 3 - L === R -> Loop terminates. Return nums[3] = 1.
TypeScript 
function findMin(nums: number[]): number {
    let l = 0;
    let r = nums.length - 1;
    
    while (l < r) {
        const mid = l + Math.floor((r - l) / 2);
        if (nums[mid] > nums[r]) {
            l = mid + 1;
        } else {
            r = mid;
        }
    }
    return nums[l];
}
Time Complexity: O(log N)
Space Complexity: O(1)

#33 Search in Rotated Sorted Array

Medium
There is an integer array nums sorted in ascending order (with distinct values). Prior to being passed to your function, nums is possibly rotated.

Given the array nums after the rotation and an integer target, return the index of target if it is in nums, or -1 if it is not in nums.

A rotated sorted array will always have at least one sorted half (either `L..mid` or `mid..R`). Determine which half is sorted by comparing `nums[L]` and `nums[mid]`.

Check if `target` lies within the sorted range. If it does, narrow the search space to that half. If it doesn't, search the other half.

Input: nums = [4, 5, 6, 7, 0, 1, 2], target = 0 Step 1: L = 0, R = 6 - mid = floor((0 + 6) / 2) = 3 - nums[mid] = nums[3] = 7 - Left half [4, 5, 6, 7] is sorted because nums[L](4) <= nums[mid](7). - Target (0) in left half? (0 >= 4 and 0 <= 7)? No. - Search right half. Set L = mid + 1 = 4. Step 2: L = 4, R = 6 - mid = floor((4 + 6) / 2) = 5 - nums[mid] = nums[5] = 1 - Right half [1, 2] is sorted because nums[mid](1) <= nums[R](2). - Target (0) in right half? (0 >= 1 and 0 <= 2)? No. - Search left half of this space. Set R = mid - 1 = 4. Step 3: L = 4, R = 4 - mid = floor((4 + 4) / 2) = 4 - nums[mid] = nums[4] = 0 - nums[mid] === target (0 === 0) -> Found! Return index 4.
TypeScript 
function searchRotated(nums: number[], target: number): number {
    let l = 0;
    let r = nums.length - 1;
    
    while (l <= r) {
        const mid = l + Math.floor((r - l) / 2);
        if (nums[mid] === target) return mid;
        
        if (nums[l] <= nums[mid]) {
            // Left half is sorted
            if (target >= nums[l] && target < nums[mid]) {
                r = mid - 1;
            } else {
                l = mid + 1;
            }
        } else {
            // Right half is sorted
            if (target > nums[mid] && target <= nums[r]) {
                l = mid + 1;
            } else {
                r = mid - 1;
            }
        }
    }
    return -1;
}
Time Complexity: O(log N)
Space Complexity: O(1)

#875 Koko Eating Bananas

Medium
Koko loves to eat bananas. There are n piles of bananas, the ith pile has piles[i] bananas. The guards have gone and will come back in h hours.

Koko can decide her bananas-per-hour eating speed of k. Each hour, she chooses some pile of bananas and eats k bananas from that pile. If the pile has less than k bananas, she eats all of them instead and will not eat any more bananas during this hour.

The speed `K` must lie in the range `1` to `max(piles)`. Since this search range is sorted, we can search it using Binary Search.

Check the hours taken to eat all piles at speed `mid`. If total hours <= `h`, Koko can eat all bananas at this speed. Record `mid` as a potential answer and narrow the range to the left (`R = mid - 1`) to search for a slower valid speed. Otherwise, search to the right (`L = mid + 1`).

Input: piles = [3, 6, 7, 11], h = 8 Search range: L = 1, R = 11 Step 1: L = 1, R = 11 - mid = floor((1 + 11) / 2) = 6 - Hours taken at speed 6: ceil(3/6) + ceil(6/6) + ceil(7/6) + ceil(11/6) = 1 + 1 + 2 + 2 = 6 hours - 6 hours <= 8 -> Valid speed. Store res = 6. Set R = mid - 1 = 5. Step 2: L = 1, R = 5 - mid = floor((1 + 5) / 2) = 3 - Hours taken at speed 3: 1 + 2 + 3 + 4 = 10 hours - 10 hours > 8 -> Speed 3 is too slow. Set L = mid + 1 = 4. Step 3: L = 4, R = 5 - mid = floor((4 + 5) / 2) = 4 - Hours taken at speed 4: 1 + 2 + 2 + 3 = 8 hours - 8 hours <= 8 -> Valid speed. Store res = 4. Set R = mid - 1 = 3. Step 4: L = 4, R = 3 - L > R -> Loop terminates. Return res = 4.
TypeScript 
function minEatingSpeed(piles: number[], h: number): number {
    let l = 1;
    let r = Math.max(...piles);
    let res = r;
    
    const canEat = (speed: number): boolean => {
        let hours = 0;
        for (const pile of piles) {
            hours += Math.ceil(pile / speed);
        }
        return hours <= h;
    };
    
    while (l <= r) {
        const mid = l + Math.floor((r - l) / 2);
        if (canEat(mid)) {
            res = mid;
            r = mid - 1;
        } else {
            l = mid + 1;
        }
    }
    return res;
}
Time Complexity: O(N * log(Max))
Space Complexity: O(1)
