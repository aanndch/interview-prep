The Monotonic Stack Paradigm

A Monotonic Stack is a standard stack that maintains elements in sorted order. If we push a value that violates the order, we pop elements until the sorting order is restored. This allows us to resolve the "next greater/smaller element" queries in O(N) overall time:

  • Decreasing Stack: For finding the next greater element. When we see a larger element than the stack top, pop and resolve all smaller keys.
  • Linear Time: Each element is pushed once and popped at most once, yielding O(N) complexity despite nested stack loops.

#20 Valid Parentheses

Easy
Given a string s containing just the characters '(', ')', '{', '}', '[' and ']', determine if the input string is valid.

An input string is valid if:
  1. Open brackets must be closed by the same type of brackets.
  2. Open brackets must be closed in the correct order.

A stack is perfect here because the last opened bracket must be the first closed one (LIFO).

Iterate through the string. When seeing an open bracket, push its matching closing bracket onto the stack. If seeing a closing bracket, pop the stack and verify it matches the current character. If stack is empty or mismatch occurs, expression is invalid.

Input: s = "()[]{}" Step 1: s[0] = '(' -> Push matching ')' onto stack. Stack: [')'] Step 2: s[1] = ')' -> Pop stack. Popped value ')' matches current ')' -> Valid. Stack: [] Step 3: s[2] = '[' -> Push matching ']' onto stack. Stack: [']'] Step 4: s[3] = ']' -> Pop stack. Popped value ']' matches current ']' -> Valid. Stack: [] Step 5: s[4] = '{' -> Push matching '}' onto stack. Stack: ['}'] Step 6: s[5] = '}' -> Pop stack. Popped value '}' matches current '}' -> Valid. Stack: [] Result: true
TypeScript 
function isValid(s: string): boolean {
    const stack: string[] = [];
    const pairs: Record<string, string> = {
        '(': ')',
        '[': ']',
        '{': '}'
    };
    
    for (const char of s) {
        if (pairs[char]) {
            stack.push(pairs[char]);
        } else {
            if (stack.pop() !== char) {
                return false;
            }
        }
    }
    return stack.length === 0;
}
Time Complexity: O(N)
Space Complexity: O(N)

#232 Implement Queue using Stacks

Easy
Implement a first-in first-out (FIFO) queue using only two stacks. The implemented queue should support all the functions of a normal queue (push, peek, pop, and empty).

A stack is LIFO. Reversing elements twice restores the FIFO order. We use two stacks: inStack to push elements, and outStack to pop/peek elements.

When popping or peeking, if outStack is empty, we transfer all elements from inStack to outStack. This reverses the elements so that the oldest elements sit at the top of outStack, ready to be read.

Operations: push(1), push(2), pop() Step 1: push(1) -> Push onto inStack. inStack: [1], outStack: [] Step 2: push(2) -> Push onto inStack. inStack: [1, 2], outStack: [] Step 3: pop() -> outStack is empty. Transfer elements: - Pop 2 from inStack, push to outStack. inStack: [1], outStack: [2] - Pop 1 from inStack, push to outStack. inStack: [], outStack: [2, 1] - Pop from outStack. Return 1. inStack: [], outStack: [2]
TypeScript 
class MyQueue {
    private inStack: number[] = [];
    private outStack: number[] = [];

    push(x: number): void {
        this.inStack.push(x);
    }

    pop(): number {
        this.transfer();
        return this.outStack.pop()!;
    }

    peek(): number {
        this.transfer();
        return this.outStack[this.outStack.length - 1];
    }

    empty(): boolean {
        return this.inStack.length === 0 && this.outStack.length === 0;
    }

    private transfer(): void {
        if (this.outStack.length === 0) {
            while (this.inStack.length > 0) {
                this.outStack.push(this.inStack.pop()!);
            }
        }
    }
}
Time Complexity: O(1) amortized
Space Complexity: O(N)

#496 Next Greater Element I

Easy
The next greater element of some element x in an array is the first greater element that is to the right of x in the same array.

You are given two distinct 0-indexed integer arrays nums1 and nums2, where nums1 is a subset of nums2. Find the next greater element for each index in nums1 from nums2.

Use a monotonic stack to resolve next greater elements for all keys in nums2. Maintain elements in decreasing order.

Iterate through nums2. When current number exceeds stack top, pop elements from stack and record their next greater element as the current number in a Map. Then map values to elements of nums1.

Input: nums1 = [4, 1, 2], nums2 = [1, 3, 4, 2] Step 1: val = 1 -> Push to stack. Stack: [1] Step 2: val = 3 -> 3 > 1. Pop 1, Map key 1 to 3. Push 3. Stack: [3] Step 3: val = 4 -> 4 > 3. Pop 3, Map key 3 to 4. Push 4. Stack: [4] Step 4: val = 2 -> 2 <= 4. Push 2. Stack: [4, 2] Step 5: Resolve nums1: - 4 is not in Map -> -1 - 1 maps to 3 - 2 is not in Map -> -1 Result: [-1, 3, -1]
TypeScript 
function nextGreaterElement(nums1: number[], nums2: number[]): number[] {
    const stack: number[] = [];
    const map = new Map<number, number>();
    
    for (const num of nums2) {
        while (stack.length > 0 && stack[stack.length - 1] < num) {
            map.set(stack.pop()!, num);
        }
        stack.push(num);
    }
    
    return nums1.map(num => map.get(num) ?? -1);
}
Time Complexity: O(N + M)
Space Complexity: O(N)

#155 Min Stack

Medium
Design a stack that supports push, pop, top, and retrieving the minimum element in constant time.

Standard stacks only provide O(N) search for minimums. To get O(1) time, use an auxiliary stack (`minStack`) that keeps track of the minimum value at each stack height.

When pushing `val`, calculate the current minimum as `min(val, minStack.top)`. Push it to `minStack`. When popping, pop from both stacks. `minStack`'s top element will always be the correct minimum.

Operations: push(-2), push(0), push(-3), getMin() Step 1: push(-2) -> valStack: [-2], minStack: [-2] Step 2: push(0) -> valStack: [-2, 0], minStack: [-2, -2] (min of 0 and -2 is -2) Step 3: push(-3) -> valStack: [-2, 0, -3], minStack: [-2, -2, -3] (min of -3 and -2 is -3) Step 4: getMin() -> Peak minStack. Return -3.
TypeScript 
class MinStack {
    private stack: number[] = [];
    private minStack: number[] = [];

    push(val: number): void {
        this.stack.push(val);
        const currentMin = this.minStack.length === 0 
            ? val 
            : Math.min(val, this.minStack[this.minStack.length - 1]);
        this.minStack.push(currentMin);
    }

    pop(): void {
        this.stack.pop();
        this.minStack.pop();
    }

    top(): number {
        return this.stack[this.stack.length - 1];
    }

    getMin(): number {
        return this.minStack[this.minStack.length - 1];
    }
}
Time Complexity: O(1)
Space Complexity: O(N)

#739 Daily Temperatures

Medium
Given an array of integers temperatures representing the daily temperatures, return an array answer such that answer[i] is the number of days you have to wait after the ith day to get a warmer temperature. If there is no future day for which this is possible, keep answer[i] == 0 instead.

Maintain indices in a monotonic decreasing stack. When we see a temperature warmer than the index stored at the stack top, we pop the index and compute the distance `i - poppedIndex`.

This is a classic Monotonic Stack application which runs in O(N) overall time.

Input: temperatures = [73, 74, 75, 71, 69, 72] Step 1: i = 0 (73) -> Stack: [0] Step 2: i = 1 (74) -> 74 > temp[0](73). Pop index 0. ans[0] = 1 - 0 = 1. Push 1. Stack: [1] Step 3: i = 2 (75) -> 75 > temp[1](74). Pop index 1. ans[1] = 2 - 1 = 1. Push 2. Stack: [2] Step 4: i = 3 (71) -> 71 <= temp[2](75). Push index 3. Stack: [2, 3] Step 5: i = 4 (69) -> 69 <= temp[3](71). Push index 4. Stack: [2, 3, 4] Step 6: i = 5 (72) -> 72 > temp[4](69). Pop index 4. ans[4] = 5 - 4 = 1. - 72 > temp[3](71). Pop index 3. ans[3] = 5 - 3 = 2. - 72 <= temp[2](75). Stop popping. Push index 5. Stack: [2, 5] Result: [1, 1, 0, 2, 1, 0]
TypeScript 
function dailyTemperatures(temperatures: number[]): number[] {
    const res: number[] = new Array(temperatures.length).fill(0);
    const stack: number[] = []; // Stores indices
    
    for (let i = 0; i < temperatures.length; i++) {
        while (stack.length > 0 && temperatures[stack[stack.length - 1]] < temperatures[i]) {
            const idx = stack.pop()!;
            res[idx] = i - idx;
        }
        stack.push(i);
    }
    return res;
}
Time Complexity: O(N)
Space Complexity: O(N)

#150 Evaluate Reverse Polish Notation

Medium
You are given an array of strings tokens that represents an arithmetic expression in a Reverse Polish Notation.

Evaluate the expression. Return an integer that represents the value of the expression.

RPN expressions are evaluated from left to right. When we see a number, we push it to the stack. When we see an operator (+, -, *, /), we pop the top two operands, evaluate the equation, and push the result back onto the stack.

Note: division operations must truncate toward zero in JS: Math.trunc(a / b).

Input: tokens = ["2", "1", "+", "3", "*"] Step 1: token = "2" -> Push to stack. Stack: [2] Step 2: token = "1" -> Push to stack. Stack: [2, 1] Step 3: token = "+" -> Pop 1 and 2. 2 + 1 = 3. Push 3. Stack: [3] Step 4: token = "3" -> Push to stack. Stack: [3, 3] Step 5: token = "*" -> Pop 3 and 3. 3 * 3 = 9. Push 9. Stack: [9] Result: 9
TypeScript 
function evalRPN(tokens: string[]): number {
    const stack: number[] = [];
    const ops: Record<string, (a: number, b: number) => number> = {
        '+': (a, b) => a + b,
        '-': (a, b) => a - b,
        '*': (a, b) => a * b,
        '/': (a, b) => Math.trunc(a / b)
    };
    
    for (const token of tokens) {
        if (ops[token]) {
            const b = stack.pop()!;
            const a = stack.pop()!;
            stack.push(ops[token](a, b));
        } else {
            stack.push(parseInt(token, 10));
        }
    }
    return stack[0];
}
Time Complexity: O(N)
Space Complexity: O(N)

#71 Simplify Path

Medium
Given a string path, which is an absolute path (starting with a slash '/') to a file or directory in a Unix-style file system, convert it to the simplified canonical path.

Split the absolute path by '/'. This results in directories, empty elements (from consecutive slashes), and special tokens '.' and '..'.

Loop through the tokens. Ignore empty names and '.'. If seeing '..', pop the stack to go up a directory level. Push any other directory names onto the stack. Finally, join the stack with '/' prefixed.

Input: path = "/a/./b/../../c/" Parsed tokens: ["a", ".", "b", "..", "..", "c"] Step 1: token = "a" -> Push. Stack: ["a"] Step 2: token = "." -> Skip. Stack: ["a"] Step 3: token = "b" -> Push. Stack: ["a", "b"] Step 4: token = ".." -> Pop "b". Stack: ["a"] Step 5: token = ".." -> Pop "a". Stack: [] Step 6: token = "c" -> Push. Stack: ["c"] Result: "/c"
TypeScript 
function simplifyPath(path: string): string {
    const stack: string[] = [];
    const parts = path.split('/');
    
    for (const part of parts) {
        if (part === '' || part === '.') {
            continue;
        }
        if (part === '..') {
            stack.pop();
        } else {
            stack.push(part);
        }
    }
    return '/' + stack.join('/');
}
Time Complexity: O(N)
Space Complexity: O(N)
