The Two Pointer Technique

Two Pointers is a technique typically used on sorted arrays or strings to resolve questions in O(N) time and **O(1) auxiliary space**. Rather than scanning elements in nested loops, we maintain two pointer indices that move relative to one another (either from both ends inward, or in tandem at different speeds).

#125 Valid Palindrome

Easy
A phrase is a palindrome if, after converting all uppercase letters into lowercase letters and removing all non-alphanumeric characters, it reads the same forward and backward. Alphanumeric characters include letters and numbers.

Given a string s, return true if it is a palindrome, or false otherwise.

A palindrome reads the same forwards and backwards. Instead of copying the string and reversing it (which takes O(N) extra space), we can check it in place.

Initialize a pointer L at the beginning and R at the end. Step inward. Skip characters that aren't letters or numbers. Compare the characters case-insensitively. If mismatch, return false.

Intuition Shift

Instead of allocating a new string to sanitize and reverse, utilize two pointers in place, skipping punctuation on the fly to guarantee O(1) space.
Input: s = "A man, a plan, a canal: Panama" L = 0 (char: "A") | R = 29 (char: "a") - L is alphanumeric, R is alphanumeric. - Lowercase compare: "a" === "a" -> Match. - Move L to 1, R to 28. L = 1 (char: " ") | R = 28 (char: "m") - L is NOT alphanumeric. Skip L. Move L to 2. L = 2 (char: "m") | R = 28 (char: "m") - Both alphanumeric. Lowercase compare: "m" === "m" -> Match. - Move L to 3, R to 27. ... (Continues until pointers meet or cross) Result: True
TypeScript 
function isPalindrome(s: string): boolean {
    let l = 0;
    let r = s.length - 1;
    
    const isAlphanumeric = (char: string) => {
        const code = char.charCodeAt(0);
        return (code >= 48 && code <= 57) || // 0-9
               (code >= 65 && code <= 90) || // A-Z
               (code >= 97 && code <= 122);  // a-z
    };
    
    while (l < r) {
        while (l < r && !isAlphanumeric(s[l])) l++;
        while (l < r && !isAlphanumeric(s[r])) r--;
        
        if (s[l].toLowerCase() !== s[r].toLowerCase()) {
            return false;
        }
        l++;
        r--;
    }
    return true;
}
Time Complexity: O(N)
Space Complexity: O(1)

#283 Move Zeroes

Easy
Given an integer array nums, move all 0's to the end of it while maintaining the relative order of the non-zero elements.

Note that you must do this in-place without making a copy of the array.

We need to push all zeroes to the end of the array without copying it. To do this, use a **slow pointer** to track where the next non-zero element should go, and a **fast pointer** to explore the array.

Every time the fast pointer spots a non-zero element, swap the values at the slow and fast pointers, and advance the slow pointer.

Input: nums = [0, 1, 0, 3, 12] Step 1: fast = 0 | nums[0] = 0 - Element is 0. - Action: Skip swap. - State: slow = 0, fast = 1 Step 2: fast = 1 | nums[1] = 1 - Element is non-zero. - Action: Swap nums[slow] and nums[fast] -> Swap nums[0] and nums[1]. - Array: [1, 0, 0, 3, 12] - State: slow = 1, fast = 2 Step 3: fast = 2 | nums[2] = 0 - Element is 0. - Action: Skip swap. - State: slow = 1, fast = 3 Step 4: fast = 3 | nums[3] = 3 - Element is non-zero. - Action: Swap nums[slow] and nums[fast] -> Swap nums[1] and nums[3]. - Array: [1, 3, 0, 0, 12] - State: slow = 2, fast = 4 Step 5: fast = 4 | nums[4] = 12 - Element is non-zero. - Action: Swap nums[slow] and nums[fast] -> Swap nums[2] and nums[4]. - Array: [1, 3, 12, 0, 0] - State: slow = 3, fast = 5 (End)
TypeScript 
function moveZeroes(nums: number[]): void {
    let slow = 0;
    for (let fast = 0; fast < nums.length; fast++) {
        if (nums[fast] !== 0) {
            // Swap elements
            const temp = nums[slow];
            nums[slow] = nums[fast];
            nums[fast] = temp;
            slow++;
        }
    }
}
Time Complexity: O(N)
Space Complexity: O(1)

#344 Reverse String

Easy
Write a function that reverses a string. The input string is given as an array of characters s.

You must do this by modifying the input array in-place with O(1) extra memory.

Reverse an array of characters in-place. We position one pointer at index 0 and another at length - 1. Swap the characters, move pointers inward, and stop when they meet or cross.

Input: s = ["h", "e", "l", "l", "o"] Step 1: L = 0 (char: "h"), R = 4 (char: "o") - Action: Swap s[0] and s[4] - Array state: ["o", "e", "l", "l", "h"] - Pointers: Move L to 1, R to 3 Step 2: L = 1 (char: "e"), R = 3 (char: "l") - Action: Swap s[1] and s[3] - Array state: ["o", "l", "l", "e", "h"] - Pointers: Move L to 2, R to 2 Step 3: L = 2, R = 2 - Action: Pointers met (L >= R), terminate loop.
TypeScript 
function reverseString(s: string[]): void {
    let l = 0;
    let r = s.length - 1;
    while (l < r) {
        const temp = s[l];
        s[l] = s[r];
        s[r] = temp;
        l++;
        r--;
    }
}
Time Complexity: O(N)
Space Complexity: O(1)

#977 Squares of a Sorted Array

Easy
Given an integer array nums sorted in non-decreasing order, return an array of the squares of each number sorted in non-decreasing order.

The array is sorted, but it contains negative values. Squaring makes negative numbers positive, which means the largest squares will actually sit at either the far-left or the far-right boundaries of the array.

Initialize one pointer at the start and another at the end. Compare the squared values. Fill the result array from back to front, placing the larger squared value at the current end of the output array.

Intuition Shift

Instead of squaring the array and re-sorting it (O(N log N)), select the largest squares from the outer margins inward (O(N)) using two pointers.
Input: nums = [-4, -1, 0, 3, 10] Initial res = [_, _, _, _, _] Step 1: L = 0 (val = -4), R = 4 (val = 10) - Compare squares: |-4|^2 = 16 vs |10|^2 = 100 - Action: 100 > 16 -> place 100 at res[4] - Pointers: Move R to 3. Array state: [_, _, _, _, 100] Step 2: L = 0 (val = -4), R = 3 (val = 3) - Compare squares: |-4|^2 = 16 vs |3|^2 = 9 - Action: 16 > 9 -> place 16 at res[3] - Pointers: Move L to 1. Array state: [_, _, _, 16, 100] Step 3: L = 1 (val = -1), R = 3 (val = 3) - Compare squares: |-1|^2 = 1 vs |3|^2 = 9 - Action: 9 > 1 -> place 9 at res[2] - Pointers: Move R to 2. Array state: [_, _, 9, 16, 100] Step 4: L = 1 (val = -1), R = 2 (val = 0) - Compare squares: |-1|^2 = 1 vs |0|^2 = 0 - Action: 1 > 0 -> place 1 at res[1] - Pointers: Move L to 2. Array state: [_, 1, 9, 16, 100] Step 5: L = 2 (val = 0), R = 2 (val = 0) - Compare squares: |0|^2 = 0 vs |0|^2 = 0 - Action: Place 0 at res[0] - Pointers: Move L to 3 (Loop terminates). Array state: [0, 1, 9, 16, 100]
TypeScript 
function sortedSquares(nums: number[]): number[] {
    const res = new Array(nums.length);
    let l = 0;
    let r = nums.length - 1;
    
    for (let i = nums.length - 1; i >= 0; i--) {
        if (Math.abs(nums[l]) > Math.abs(nums[r])) {
            res[i] = nums[l] * nums[l];
            l++;
        } else {
            res[i] = nums[r] * nums[r];
            r--;
        }
    }
    return res;
}
Time Complexity: O(N)
Space Complexity: O(N) (for output array)

#11 Container With Most Water

Medium
You are given an integer array height of length n. There are n vertical lines drawn such that the two endpoints of the ith line are (i, 0) and (i, height[i]).

Find two lines that together with the x-axis form a container, such that the container contains the most water.

Return the maximum amount of water a container can store.

The amount of water trapped is limited by the shorter of the two boundaries: Area = min(height[L], height[R]) * (R - L).

We position our pointers at the far ends (L=0 and R=height.length-1) to maximize the width. To find a larger area, we must search for taller lines. Since the current height is bottlenecked by the smaller height, moving the taller pointer will never increase the area. Therefore, we always shift the pointer that has the smaller height.

Input: height = [1, 8, 6, 2, 5, 4, 8, 3, 7] Step 1: L = 0 (h: 1) | R = 8 (h: 7) - Width: R - L = 8 - 0 = 8 - Current Area: min(1, 7) * 8 = 8 - Max Area: max(0, 8) = 8 - Action: height[L] (1) < height[R] (7) -> Move L to 1 Step 2: L = 1 (h: 8) | R = 8 (h: 7) - Width: 8 - 1 = 7 - Current Area: min(8, 7) * 7 = 49 - Max Area: max(8, 49) = 49 - Action: height[R] (7) < height[L] (8) -> Move R to 7 Step 3: L = 1 (h: 8) | R = 7 (h: 3) - Width: 7 - 1 = 6 - Current Area: min(8, 3) * 6 = 18 - Max Area: max(49, 18) = 49 - Action: height[R] (3) < height[L] (8) -> Move R to 6 ... (Repeats until pointers meet) Max Area Result: 49
TypeScript 
function maxArea(height: number[]): number {
    let l = 0;
    let r = height.length - 1;
    let maxVal = 0;
    
    while (l < r) {
        const area = Math.min(height[l], height[r]) * (r - l);
        maxVal = Math.max(maxVal, area);
        
        if (height[l] < height[r]) {
            l++;
        } else {
            r--;
        }
    }
    return maxVal;
}
Time Complexity: O(N)
Space Complexity: O(1)

#15 3Sum

Medium
Given an integer array nums, return all the triplets [nums[i], nums[j], nums[k]] such that i != j, i != k, and j != k, and nums[i] + nums[j] + nums[k] == 0.

Notice that the solution set must not contain duplicate triplets.

We want to find all unique triplets that add up to 0. First, sort the array (which takes O(N log N)).

Iterate through the array, locking in the current element nums[i] as the first number. To find the remaining two numbers, run a standard Two-Pointer scan on the array to the right of index i. To avoid duplicate triplets, skip identical values for i, L, and R.

Intuition Shift

Sort first, then convert a 3-variable problem into a nested 2-pointer scan. Skipped duplicate values dynamically to guarantee unique triplet outputs.
Input: nums = [-1, 0, 1, 2, -1, -4] Sorted array: [-4, -1, -1, 0, 1, 2] Pass 1: i = 0 | nums[0] = -4 - Lock in first element = -4. Target sum for remaining = 4. - L = 1 (val: -1) | R = 5 (val: 2) -> sum = 1 < 4 -> Move L to 2 - L = 2 (val: -1) | R = 5 (val: 2) -> sum = 1 < 4 -> Move L to 3 - L = 3 (val: 0) | R = 5 (val: 2) -> sum = 2 < 4 -> Move L to 4 - L = 4 (val: 1) | R = 5 (val: 2) -> sum = 3 < 4 -> Move L to 5 (L >= R, terminates) Pass 2: i = 1 | nums[1] = -1 - Lock in first element = -1. Target sum for remaining = 1. - L = 2 (val: -1) | R = 5 (val: 2) -> sum = 1 === 1 -> Found triplet [-1, -1, 2]. - Action: Shift pointers: L to 3, R to 4 (skipping duplicate values) - L = 3 (val: 0) | R = 4 (val: 1) -> sum = 1 === 1 -> Found triplet [-1, 0, 1]. - Action: Shift pointers: L to 4, R to 3 (L >= R, terminates) Pass 3: i = 2 | nums[2] = -1 - nums[2] === nums[1] -> Skip iteration to prevent duplicate triplets.
TypeScript 
function threeSum(nums: number[]): number[][] {
    nums.sort((a, b) => a - b);
    const res: number[][] = [];
    
    for (let i = 0; i < nums.length - 2; i++) {
        if (i > 0 && nums[i] === nums[i - 1]) continue;
        
        let l = i + 1;
        let r = nums.length - 1;
        
        while (l < r) {
            const sum = nums[i] + nums[l] + nums[r];
            if (sum === 0) {
                res.push([nums[i], nums[l], nums[r]]);
                while (l < r && nums[l] === nums[l + 1]) l++;
                while (l < r && nums[r] === nums[r - 1]) r--;
                l++;
                r--;
            } else if (sum < 0) {
                l++;
            } else {
                r--;
            }
        }
    }
    return res;
}
Time Complexity: O(N²)
Space Complexity: O(1) (excluding sorting storage)

#42 Trapping Rain Water

Medium
Given n non-negative integers representing an elevation map where the width of each bar is 1, compute how much water it can trap after raining.

The water level at any index i is determined by the minimum of the highest walls to its left and right, minus the height of the bar at i itself: Water = min(maxLeft, maxRight) - height[i].

To do this in O(N) time and **O(1) space**, place pointers at the far ends (L=0, R=height.length-1) and maintain leftMax and rightMax. Move the pointer pointing to the smaller bar, updating the max boundaries and accumulating trapped water on the fly.

Input: height = [0, 1, 0, 2, 1, 0, 1, 3, 2, 1, 2, 1] L = 0 (h: 0) | R = 11 (h: 1) | leftMax = 0 | rightMax = 0 - height[L] < height[R] (0 < 1) -> Process L - Update leftMax = max(0, 0) = 0 - Water += leftMax - height[L] = 0 - 0 = 0 - Action: Move L to 1 L = 1 (h: 1) | R = 11 (h: 1) | leftMax = 0 | rightMax = 0 - height[L] <= height[R] (1 <= 1) -> Process L - Update leftMax = max(0, 1) = 1 - Water += leftMax - height[L] = 1 - 1 = 0 - Action: Move L to 2 L = 2 (h: 0) | R = 11 (h: 1) | leftMax = 1 | rightMax = 0 - height[L] < height[R] (0 < 1) -> Process L - Water += leftMax - height[L] = 1 - 0 = 1 - Action: Move L to 3 ... (Repeats until pointers meet) Total Water Trapped: 6
TypeScript 
function trap(height: number[]): number {
    let l = 0;
    let r = height.length - 1;
    let leftMax = 0;
    let rightMax = 0;
    let res = 0;
    
    while (l < r) {
        if (height[l] < height[r]) {
            if (height[l] >= leftMax) {
                leftMax = height[l];
            } else {
                res += leftMax - height[l];
            }
            l++;
        } else {
            if (height[r] >= rightMax) {
                rightMax = height[r];
            } else {
                res += rightMax - height[r];
            }
            r--;
        }
    }
    return res;
}
Time Complexity: O(N)
Space Complexity: O(1)

#16 3Sum Closest

Medium
Given an integer array nums of length n and an integer target, find three integers in nums such that the sum is closest to target.

Return the sum of the three integers. You may assume that each input would have exactly one solution.

Similar to 3Sum, sort the array first. Loop through, fixing the first element nums[i]. Use a left and right pointer to scan the remainder of the array.

Calculate the sum of the triplet. Track the sum that has the absolute minimum difference from the target. Move the pointers closer to the target (L++ if sum is smaller than target, R-- if larger).

Input: nums = [-1, 2, 1, -4], target = 1 Sorted array: [-4, -1, 1, 2] Pass 1: i = 0 | nums[0] = -4 - L = 1 (val: -1) | R = 3 (val: 2) - Sum = -4 + (-1) + 2 = -3 - Difference: |1 - (-3)| = 4. Update closest = -3. - sum < target (-3 < 1) -> Move L to 2 - L = 2 (val: 1) | R = 3 (val: 2) - Sum = -4 + 1 + 2 = -1 - Difference: |1 - (-1)| = 2. Update closest = -1. - sum < target (-1 < 1) -> Move L to 3 (L >= R, terminates) Pass 2: i = 1 | nums[1] = -1 - L = 2 (val: 1) | R = 3 (val: 2) - Sum = -1 + 1 + 2 = 2 - Difference: |1 - 2| = 1. Update closest = 2. - sum > target (2 > 1) -> Move R to 2 (L >= R, terminates) Result Closest Sum: 2
TypeScript 
function threeSumClosest(nums: number[], target: number): number {
    nums.sort((a, b) => a - b);
    let closest = nums[0] + nums[1] + nums[2];
    
    for (let i = 0; i < nums.length - 2; i++) {
        let l = i + 1;
        let r = nums.length - 1;
        
        while (l < r) {
            const sum = nums[i] + nums[l] + nums[r];
            if (Math.abs(target - sum) < Math.abs(target - closest)) {
                closest = sum;
            }
            
            if (sum < target) {
                l++;
            } else if (sum > target) {
                r--;
            } else {
                return sum; // exact match
            }
        }
    }
    return closest;
}
Time Complexity: O(N²)
Space Complexity: O(1)
