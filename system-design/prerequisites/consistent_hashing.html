Consistent Hashing

Prerequisites / Concepts — Distributed Systems Foundations

Core Concept

**Consistent Hashing** is a key routing technique that maps both database/cache nodes and key hashes to a circular 360-degree numeric space. When nodes are added or removed, this ring structure guarantees that only a tiny fraction of keys ($\approx 1/N$) are migrated, shielding clusters from cascading cache misses.

1. Modulo Hashing vs. Consistent Hashing

Traditional cluster hashing distributes traffic using the formula hash(key) % N:

  • The Problem: When scaling a cluster, adding a new node changes the modulo denominator N. This re-hashes and remaps nearly all keys to completely new node servers.
  • Consistent Hashing: Nodes and keys are hashed onto the same circular space. Adding a node only intercepts keys in its immediate counter-clockwise vicinity, leaving other mapping states untouched.

2. Ring Routing & Data Locality

How data is located on the circular token ring space:

  • Nodes and keys are mapped via MD5 or SHA-1 hashes (e.g. 0 to $2^{32} - 1$).
  • Routing Rule: Each key is routed to the first node encountered by moving clockwise from the key's position on the ring.

3. Virtual Nodes (VNodes)

Physical servers mapped to multiple, randomized coordinates on the ring:

  • Solving Data Skew: Placing only physical servers on a ring creates uneven intervals, leading to hotspots.
  • Mechanism: One machine (e.g. Server A) represents 100 virtual nodes (e.g. A-1, A-2... on the ring). This interlaces nodes across the ring, balancing data distribution.
  • Hardware Heterogeneity: Faster servers can be assigned more VNodes to handle a larger proportion of keys.

4. Real-world Deployments

Where this pattern is deployed in production architectures:

  • Consistent Hashing Clients: Memcached/Redis client libraries map keys locally to select cache partitions.
  • Distributed Ring DBs: Apache Cassandra and AWS DynamoDB map partition keys across a DHT (Distributed Hash Table) ring of node instances.

How to Answer in an Interview

When designing distributed cache clusters or partitioning, consistent hashing is a must-mention:

"To prevent cascading cache misses when scaling node pools, I will avoid modulo hashing and implement a consistent hashing ring. We will map both cache keys and servers onto the same 360-degree hash space, routing keys to the next clockwise node. To prevent data hotspots and balance load across heterogeneous machines, I will use virtual nodes (VNodes), mapping each physical server to 100+ virtual coordinates interlaced on the ring."
