FND Prerequisites — study these first
CONCEPT 1.1
Scalability
Ability of a system to handle increasing load. Vertical scaling (scale up) vs. horizontal scaling (scale out) and database sharding.
CONCEPT 1.2
Reliability & Fault Tolerance
Strategies for handling inevitable failures: replication, retry & backoff, circuit breakers, rate limiting, graceful degradation, and idempotency.
CONCEPT 1.3
Performance
Optimizing latency and throughput via caching, database indexing, connection pooling, data compression, batching network calls, and gRPC/WebSockets.
CONCEPT 1.4
CAP Theorem
The trade-off between Consistency, Availability, and Partition Tolerance in distributed data stores. Choosing between CP and AP systems.
CONCEPT 1.5
Caching
Storing copies of data in high-speed RAM (Redis/Memcached). Caching strategies (TTL, Write-Through, Write-Behind, Cache-Aside) and eviction policies.
CONCEPT 1.6
Load Balancing
Distributing incoming traffic evenly across servers. L4 (transport layer) vs. L7 (application layer) routing, round robin, least connections, and IP hashing.
CONCEPT 1.7
Databases
Comparing Relational (SQL) databases with ACID properties and Non-Relational (NoSQL) databases with the BASE model for different scaling needs.
CONCEPT 1.8
Sharding & Partitioning
Horizontal partitioning of database tables. Key sharding strategies (hash-based, range-based, directory-based) and hot spot rebalancing challenges.
CONCEPT 1.9
Consistent Hashing
Minimizing data movement when scaling node pools. Ring-based mapping and virtual nodes used in distributed caches and databases like Cassandra and DynamoDB.
CONCEPT 1.10
Replication
Copying data across nodes for durability and availability. Primary-Replica vs. Multi-Master architectures, and synchronous vs. asynchronous updates.
CONCEPT 1.11
Indexing
Optimizing query performance via B-tree, Hash, Inverted, Trie, and Bitmap indexes. Composite/covering indexes and write performance trade-offs.
CONCEPT 1.12
Message Queues
Decoupling microservices via asynchronous message brokers (RabbitMQ, Kafka, SQS). Delivery semantics (at-least-once) and backpressure management.
CONCEPT 1.13
Observability & Monitoring
Diagnosing systems using the three pillars: Metrics (Prometheus), Logs (ELK), and Distributed Tracing (Jaeger) along with RED and USE methods.
CONCEPT 1.14
Data Pipelines & Streaming
Processing data in batches (Spark/Hadoop) or streams (Kafka/Flink). Lambda vs. Kappa architectures for real-time analytics.
CONCEPT 1.15
Security & Authentication
Securing services and identities. OAuth 2.0/OIDC flows, JWT vs. Session cookies, CORS preflights, CSRF double-submits, and API gateway rules.
CONCEPT 1.16
Client-Server Sync & Fetching
Data query paradigms (REST, GraphQL, gRPC) and UI synchronization via optimistic updates, offline store sync, and HTTP caching.
